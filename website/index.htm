3D Rendering Techniques and Algorithms - winter 2004

"All it takes is for the rendered image to look right." - Jim Blinn (1983)

"It is important that we can understand the true physical behaviour so when we make approximations we recognize the consequences." - Donald P. Greenberg (1988)

"Nothing is less real than realism.  Details are confusing.  It is only by selection, by elimination, by emphasis that we get at the real meaning of things." - Georgia O'Keeffe (1922)

"We all know that art is not truth.  Art is a lie that makes us realize the truth." - Pablo Picasso

This course is an exploration of realistically rendering 3D scenes by computer.  Realistically rendering 3D scenes is a very challenging endeavor.  It requires an understanding of a variety of disciplines - mathematics, geometry, physics, optics, colour science, and perceptual psychology - and the reduction of this understanding to a computationally tractable algorithmic form.

There are two basic parts to the problem:

what do we see;
and how does it look.

To determine what we see, we need to:

build a mathematical/algorithmic representation of the geometry of the scene;
build a mathematical model of the observer;
project the description of the scene into the viewing space of the observer, determining which parts a visible, and which are obscured.

To determine how it looks, we need to:

build a mathematical/algorithmic model of the materials and surface finishes of all the objects in the scene;
build a mathematical/algorithmic model of the lighting in the scene;
model the physics/optics of the interaction of light with the geometry and surface finishes of the scene;
filter the result through the perceptual mechanism of the observer to determine how to reproduce the perceptual experience of viewing the scene;
and factor in the technology for image display/reproduction.

Taking all of this into account, and accounting for the fact that computational resources are not infinite, but the complexity of the scenes we wish to render is essentially infinite, we have a very interesting problem to solve.

Course Objectives

Upon completion of this course, each student should have sufficient understanding of realistic image generation to be a productive participant in any project that has a 3D rendering component.  This would range from hardware design, through graphics API implementation, through realistic image production for film or video.

Homework and Term Project

There will be weekly programming assignments supporting the term project of writing a realistic 3D renderer. The term project will be extending the renderer written in the first part of the course to include advanced rendering features of the student's choice.

Prerequisites

It is assumed the student can skillfully program in some computer language - i.e. C, C++, or Java, VB, Fortan, etc.  All the libraries I post will be in Java (see Assignments).  A basic knowledge of linear algebra is helpful.  While Java is my language of choice for the class, rendering as discussed in this class, is primarily a computational exercise that is easily written in any language.  I've chosen Java as the language of choice because the object model makes it easy to build aan extensible framework supporting student extensions.
